# AK/SK Signature Specification (Gateway Auth v1)

## 1. Scope

This specification defines one canonical AK/SK signing contract for all gateway entry paths.

## 2. Required Request Fields

All fields are required and transmitted as headers or transport-equivalent metadata:

| Field | Description |
| --- | --- |
| `ak` | Access key identifier (opaque, non-semantic) |
| `tenant_id` | Tenant identifier |
| `client_id` | Client application identifier |
| `timestamp` | UTC epoch seconds generated by client |
| `nonce` | Single-use random token per auth attempt |
| `session_id` | Client session identifier for correlation |
| `signature` | Base64(HMAC-SHA256(canonical_string, SK)) |

## 3. Canonical String

Canonical field order is fixed:

1. `ak`
2. `tenant_id`
3. `client_id`
4. `timestamp`
5. `nonce`
6. `session_id`

Canonical format:

```text
ak:{ak}
tenant_id:{tenant_id}
client_id:{client_id}
timestamp:{timestamp}
nonce:{nonce}
session_id:{session_id}
```

Rules:

- UTF-8 encoding.
- Unix line ending (`\n`) between fields.
- No trailing whitespace, no trailing newline.
- Empty or missing values are invalid and rejected before signature validation.

## 4. Timestamp and Skew Validation

- Gateway compares `timestamp` with server UTC clock.
- Accepted absolute skew window default: `300s` (configurable).
- Requests outside skew window are rejected with `AUTH_V1_TIMESTAMP_OUT_OF_WINDOW`.
- Reconnect and renewal requests use the same skew rule.

## 5. Anti-Replay Policy

Replay key namespace:

```text
auth:replay:v1:{tenant_id}:{client_id}:{ak}:{nonce}
```

Policy:

- The first seen key is accepted and stored with replay TTL.
- Duplicate key within replay TTL is rejected as replay.
- Default replay TTL: `300s` (configurable).

## 6. Redaction Rules

Never log or return:

- Raw `signature`
- Any part of `sk`
- `ak` substrings beyond masked format

Safe logging:

- `tenant_id`, `client_id`, `trace_id`, `session_id`, `error_code`, `debug_id`
- Masked AK: `ak_****{last4}`

## 7. Principal Contract

On successful authentication, gateway principal exposes:

- `tenant_id`
- `client_id`
- `trace_id`
- `session_id`
- `credential_state`
- `issued_at`
