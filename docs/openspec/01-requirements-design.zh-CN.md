# 1. 文档目的

本文件定义 `OpenCode Skill Bridge` 下一里程碑（`v1.1`）的需求基线，覆盖功能需求、非功能需求、验收标准与追踪关系。  
本需求文档与以下 OpenSpec 文档配套使用：

- `02-architecture-design.zh-CN.md`（架构设计）
- `03-solution-design.zh-CN.md`（方案设计）
- `04-test-cases.zh-CN.md`（测试用例）

# 2. 背景与问题陈述

`v1.0` 已完成核心闭环：

- Slash 触发技能会话
- 插件长连接与网关认证
- Skill 持久化与会话历史
- 选段回传 IM
- 可靠性与可观测性基线

下一阶段主要问题：

- 仅 Web/PC 形态完成验证，移动端能力尚未对齐。
- 缺少租户级权限与配额治理能力。
- 缺少面向运营/审计角色的统一查询与导出能力。

# 3. 范围定义

## 3.1 In Scope

- Android/iPhone/Harmony 的技能触发、会话、回传能力对齐。
- 租户级 RBAC 权限策略。
- 租户级调用配额和限流。
- 审计看板查询与导出能力。
- 兼容性与治理策略（`contract_version`、硬门禁、豁免机制）。

## 3.2 Out of Scope

- 多 Agent 编排与工具市场能力。
- 跨地域双活部署。
- 大规模商业化计费系统。

# 4. 术语

- `Skill`：由 OpenCode 提供的 AI 能力会话入口。
- `Gateway`：负责鉴权、流控、协议桥接与路由的服务。
- `Skill Service`：负责会话持久化、历史查询、回传 IM 的服务。
- `Tenant`：企业租户隔离单元。
- `contract_version`：插件/网关边界事件的契约版本字段。
- `Hard Gate`：发布阻断门禁，未通过即不可发布。

# 6. 详细需求

## FR-001 多端技能触发一致性

- 描述：Android/iPhone/Harmony 支持与 Web/PC 一致的 Slash 触发与技能入口。
- 验收标准：
  - 输入 `/` 时可见 `SKILL` 选择器且包含 `Local OpenCode`。
  - 触发面板字段一致：会话上下文、问题输入、提交按钮。
  - 触发失败时返回可操作错误码。

## FR-002 多端会话连续性一致性

- 描述：移动端支持会话页打开、增量回包展示、断线重连后恢复。
- 验收标准：
  - 会话页与聊天卡片状态同步。
  - 重连后消息序列不重复、不倒序。
  - 失败场景返回 `reason_code + next_action`。

## FR-003 多端回传 IM 一致性

- 描述：移动端可选择 AI 输出片段并回传 IM，保留相关性。
- 验收标准：
  - 支持单段选取、预览确认、提交结果提示。
  - 成功返回 IM 消息 ID 与关联键。
  - 失败支持保留草稿并一键重试。

## FR-004 租户级 RBAC 权限控制

- 描述：提供按租户/角色/能力的访问控制（触发、会话、回传）。
- 验收标准：
  - 支持策略配置：`allow`、`deny`、`scope`。
  - 未授权请求在网关层拒绝并记录审计事件。
  - 权限变更在 SLA 时间内生效（<= 60s）。

## FR-005 租户级配额与限流

- 描述：对技能调用设置租户维度配额（分钟、小时、日）与瞬时限流。
- 验收标准：
  - 支持配额策略配置与查询。
  - 超限返回稳定错误码和重试建议。
  - 配额统计可按租户/客户端维度聚合。

## FR-006 审计看板查询能力

- 描述：面向运营角色提供技能调用与回传行为可查询看板。
- 验收标准：
  - 支持按租户、时间、用户、会话、结果状态过滤。
  - 支持分页与排序。
  - 支持展示全链路追踪字段（`trace_id`、`session_id`）。

## FR-007 审计事件导出能力

- 描述：支持审计记录导出供审计留档。
- 验收标准：
  - 支持 CSV 导出。
  - 导出范围受权限与时间窗限制。
  - 导出任务可追踪状态（提交、处理中、完成、失败）。

## FR-008 租户治理配置中心

- 描述：集中管理权限策略、配额策略、客户端开关。
- 验收标准：
  - 配置具备版本号和变更审计。
  - 支持灰度生效（按租户/客户端）。
  - 回滚操作可在 5 分钟内完成。

## FR-009 边界契约版本兼容治理

- 描述：插件与网关边界事件必须带 `contract_version` 并遵循兼容策略。
- 验收标准：
  - 所有关键事件包含 `contract_version`。
  - 仅允许新增字段/事件，不允许破坏性删除。
  - 兼容回归测试纳入发布门禁。

## FR-010 发布门禁与豁免闭环

- 描述：发布前必须通过硬门禁；豁免需审批、到期与追踪。
- 验收标准：
  - 标准命令可一键执行完整校验。
  - 豁免记录包含 owner/approver/expiration。
  - 过期豁免自动失效并阻断发布。

# 7. 用户旅程（关键场景）

- 场景 A：移动端用户 Slash 触发技能并继续多轮会话。
- 场景 B：用户选择模型输出片段并回传 IM。
- 场景 C：管理员配置租户权限与配额并查看效果。
- 场景 D：运营人员查询审计数据并导出留档。
- 场景 E：发布前执行硬门禁并处理豁免。

# 8. 约束与依赖

- 技术栈约束：JDK 21、Spring Boot 3.4.6、MyBatis、MySQL 5.7、TS 插件/前端。
- 安全约束：AK/SK 鉴权前置，敏感字段禁止明文日志。
- 兼容约束：既有 IM 消息模型与网关契约不可破坏。
- 依赖系统：IM 发送接口、网关鉴权链路、Skill 持久化服务。

# 9. 风险与缓解

| 风险 | 影响 | 缓解策略 |
|---|---|---|
| 多端能力实现差异 | 行为不一致，回归成本高 | 统一协议契约与跨端回归用例 |
| 配额策略误配置 | 误封或放量导致业务异常 | 增加灰度发布与回滚机制 |
| 审计数据量增长 | 查询与导出性能下降 | 分区索引、异步导出、冷热分层 |
| 豁免滥用 | 质量门禁失效 | 强制到期、审批链与审计追责 |
