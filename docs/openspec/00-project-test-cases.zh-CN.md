# 1. 文档目标

本测试文档用于覆盖 OpenCode Skill Bridge 全项目能力，确保“需求-设计-实现-发布”闭环质量。  
覆盖范围包含已交付能力（v1.0）与后续规划能力（多端、治理、审计、硬化）。

# 2. 覆盖范围与覆盖口径

## 2.1 需求覆盖范围

- 功能需求：`FR-001 ~ FR-010`
- 性能需求：`NFR-PERF-01 ~ NFR-PERF-04`
- 安全需求：`NFR-SEC-01 ~ NFR-SEC-05`
- 一致性需求：`NFR-CONS-01 ~ NFR-CONS-04`

## 2.2 覆盖率定义

- 需求覆盖率 = 已映射测试点的需求数 / 全量需求数
- 用例执行覆盖率 = 已执行用例数 / 全量用例数
- 异常路径覆盖率 = 已覆盖异常分支数 / 全量异常分支数

目标：三项指标均为 `100%`。

# 3. 测试策略

## 3.1 功能测试

- 黑盒测试：面向接口和用户行为，验证业务输出是否符合规格。
- 白盒测试：针对关键逻辑分支（鉴权、配额、幂等、重试）进行分支覆盖。
- 端到端测试：覆盖“触发 -> 会话 -> 回传 -> 审计”全链路。

## 3.2 边界值测试

- 输入边界：空值、超长、非法字符、极限分页、时间窗口边界。
- 状态边界：状态流转终态回退、重复提交、重复事件。
- 资源边界：连接数上限、配额临界值、队列堆积临界值。

## 3.3 异常中断测试

- 网络中断：客户端断网、服务抖动、下游 IM 超时。
- 组件中断：Redis 不可用、MQ 堆积、数据库主从切换。
- 发布中断：Hard Gate 失败、豁免过期、策略发布回滚失败。

## 3.4 安全性测试

- 鉴权与授权：AK/SK、Token、RBAC 拒绝路径。
- 数据安全：敏感字段脱敏、越权访问、注入攻击、防重放。
- 审计安全：关键操作必须留痕，不可篡改查询条件。

## 3.5 自动化策略

- 接口自动化：REST + WebSocket 协议测试。
- 回归自动化：主路径 + 高风险异常路径 nightly 执行。
- 压测自动化：峰值触发、流式消息、导出任务三类压测。

# 4. 测试环境与数据准备

## 4.1 环境

- DEV：功能联调与白盒调试。
- STAGING：发布前全链路回归、压测与故障演练。

## 4.2 测试数据

- 租户：标准租户、高配租户、受限租户各 1 个。
- 角色：`admin`、`operator`、`member` 各 1 组。
- 会话：普通会话、长会话、异常中断会话样本各 100+。
- 回传样本：正常文本、超长文本、非法文本、空文本。

# 9. 缺陷分级与处理策略

| 等级 | 定义 | 处理要求 |

| S3 | 一般问题，不影响主链路 | 可排入后续迭代 |

| 层级 | 覆盖内容 | 工具建议 |
|---|---|---|
| API 自动化 | REST 状态码、返回体、幂等与异常 | RestAssured / Postman / Newman |
| WS 自动化 | 握手、事件序号、断线恢复 | 自研 WS Test Harness |
| 集成测试 | Gateway/Skill/Policy/Audit 协作 | Testcontainers + CI |
| 压测 | 触发QPS、流式事件、导出任务 | k6 / JMeter |
| 安全测试 | 注入、越权、重放、脱敏 | OWASP ZAP + 自定义脚本 |
