# SOL-1. 目标与范围

本方案用于落地以下目标：

- 补齐多端能力（FR-001..FR-003）
- 建立租户治理（FR-004..FR-005, FR-008）
- 建设审计运营能力（FR-006..FR-007）
- 固化契约与发布治理（FR-009..FR-010）

# SOL-2. 约束条件

- 后端技术栈不变：JDK 21 + Spring Boot 3.4.6 + MyBatis + MySQL 5.7。
- 插件/客户端边界协议延续 `contract_version`。
- 不引入跨区域基础设施改造。

# SOL-3. 备选方案对比

| 方案 | 描述 | 优点 | 缺点 | 结论 |
|---|---|---|---|---|
| A | 仅在 Gateway 实现权限/配额/审计 | 改造集中 | 审计扩展与导出压力集中 | 否 |
| B | Gateway 治理 + Skill Service 审计存储导出 | 职责清晰，可扩展性好 | 需要跨服务契约同步 | 是 |
| C | 全事件驱动异步治理 | 扩展性高 | 架构复杂，落地周期长 | 否 |

选型结论：采用 **方案 B**。

# SOL-4. 总体交付策略

- 增量交付，不中断当前主链路。
- 先网关治理基础，再接入多端，再上线审计导出。
- 每个阶段都可独立回滚。

# SOL-5. 分项设计

## SOL-5.1 多端能力对齐（FR-001..FR-003）

- 统一客户端协议适配层（Android/iOS/Harmony）。
- 会话页状态机统一：
  - `pending -> in_progress -> completed|failed`
- 回传流程统一：
  - 选段 -> 预览 -> 提交 -> 结果确认。

交付物：

- 多端 SDK 适配接口定义
- 兼容回归测试集
- 客户端错误码映射表

## SOL-5.2 权限与配额治理（FR-004..FR-005）

- 在 Gateway 引入统一治理拦截器。
- 权限策略与配额策略分别模块化：
  - `AuthzPolicyEngine`
  - `QuotaGuard`
- 执行链顺序固定：鉴权 -> RBAC -> 配额 -> 业务处理。

交付物：

- 策略 CRUD API
- 拦截器与缓存刷新机制
- 超限/拒绝错误码规范

## SOL-5.3 审计看板与导出（FR-006..FR-007）

- 主链路写入审计事件（异步）。
- 查询 API 支持过滤、分页、排序。
- 导出通过任务中心异步生成 CSV。

交付物：

- 审计事件模型与索引
- 审计查询 API
- 审计导出 worker

## SOL-5.4 配置治理中心（FR-008）

- 策略版本化与 checksum 校验。
- 支持租户灰度。
- 支持一键回滚到上一版本。

交付物：

- 配置中心接口
- 版本发布与回滚操作台
- 审计变更记录

## SOL-5.5 契约与发布治理（FR-009..FR-010）

- 边界契约继续采用增量兼容策略。
- 发布门禁整合：
  - 插件 no-drift
  - host integration
  - CLI real-chain
  - gateway auth/resume regression
- 豁免机制：
  - 明确 owner/approver/expiration
  - 到期自动阻断

交付物：

- 契约兼容检查脚本
- 门禁流水线配置
- 豁免管理模板与审批流程

# SOL-6. 非功能落地方案

## SOL-6.1 性能

- 策略缓存本地化 + 定时拉取 + 变更推送。
- 审计导出异步化，避免大查询阻塞。

## SOL-6.2 可用性

- 网关实例无状态化扩容。
- 审计导出任务支持重试与断点续跑。

## SOL-6.3 安全与合规

- 审计导出权限分级。
- 导出文件带过期时间与访问控制。
- 关键操作记录审计事件。

## SOL-6.4 可观测性

- 统一日志字段和告警维度。
- 关键 SLA 指标：
  - trigger_accept_latency
  - authz_reject_rate
  - quota_reject_rate
  - export_job_success_rate

# SOL-7. 里程碑计划（建议）

| 里程碑 | 目标 | 预估 |
|---|---|---|
| M1 | 网关治理基础（RBAC + Quota） | 2 周 |
| M2 | 多端能力对齐（触发/会话/回传） | 3 周 |
| M3 | 审计查询与导出 | 2 周 |
| M4 | 契约门禁与灰度上线 | 1 周 |

# SOL-8. 回滚与应急预案

- 策略层故障：回滚至上个策略版本，恢复默认放行/限流兜底策略。
- 审计层故障：临时降级为仅日志落盘，导出功能暂时关闭。
- 客户端兼容问题：按客户端维度关闭新能力开关。

# SOL-9. 风险与依赖

| 项目 | 风险 | 缓解 |
|---|---|---|
| 多端 SDK | 平台差异导致行为不一致 | 统一状态机和契约测试 |
| 策略缓存 | 配置不一致导致误判 | 版本校验 + 校验和校验 |
| 导出任务 | 高峰期排队延迟 | 优先级队列 + 限流 |
| 治理门禁 | 豁免滥用 | 到期强制失效 + 审批审计 |
