## Summary

Phase 01.1 is an architecture correction gate, not a feature-expansion phase. Planning should focus on proving that the bridge runtime is plugin-first and that plugin mode plus CLI debug mode are thin adapters over one shared bridge core. If either mode implements separate business logic, this phase should be treated as failed due to architecture drift.

The existing Java `pc-agent` module is currently an auth/credential semantics reference, while `gateway` already enforces AUTH_V1 contracts through `AuthService`, HTTP/WS interceptors, and doc-sync tests. The safest path is to port those semantics into plugin-side contracts first, then build host and CLI adapters on top of the same core runtime, and finally remove Java runtime from main execution path.

**Primary recommendation:** Plan 01.1 around one shared bridge core plus two adapters (`host-plugin`, `cli-debug`) with mandatory compatibility gates against existing AUTH_V1 docs/tests before any legacy code removal.

## Recommended Architecture Slices for This Phase

### Slice A: Shared Bridge Core (single source of runtime truth)
- Responsibility: lifecycle engine, reconnect state machine, session context, error/health events.
- Boundary: no host UI SDK code and no CLI parsing code.
- Must own protocol conversion and gateway auth request composition.

### Slice B: Gateway Contract Adapter (AUTH_V1 compatibility)
- Responsibility: build canonical auth payload exactly matching docs and gateway expectations.
- Inputs: `ak`, `tenant_id`, `client_id`, `timestamp`, `nonce`, `session_id`, `signature`, `trace_id`.
- Outputs: deterministic handling of AUTH_V1 error envelope/WS close mapping.
- Constraint source: `docs/auth/aksk-signature-spec.md`, `docs/auth/auth-error-contract.md`, and gateway tests.

### Slice C: Credential Security Adapter
- Responsibility: secret reference read/write and typed error mapping.
- Required semantic parity with current Java references:
  - `NOT_FOUND`, `ACCESS_DENIED`, `CORRUPT_ENTRY`, `INVALID_INPUT`
  - no secret leakage in string/log output
  - DPAPI-equivalent OS user-scope protection
- Reference source: current `pc-agent` auth/config classes and tests.

### Slice D: Host Plugin Adapter
- Responsibility: host lifecycle binding (`init/start/stop/dispose`) and event-driven host contract.
- Constraint: must delegate bridge logic to Slice A, not reimplement bridge/auth logic.

### Slice E: CLI Debug Adapter
- Responsibility: command entrypoint and runtime wiring for joint debug without client.
- Constraint: must call same shared core and run real chain, not mocks.

### Slice F: Drift Guard Layer
- Responsibility: enforce architecture invariants via tests/check scripts:
  - dual mode imports same core runtime
  - protocol conversion only in bridge/core layer
  - AUTH_V1 contract snapshots stay in sync

## Migration Risks and Mitigations

| Risk | Impact | Mitigation | Exit Signal |
|---|---|---|---|
| Plugin host SDK/runtime details are unknown at planning time | Wrong scaffolding; rework in Wave 2 | Add Wave 0 discovery checkpoint for host lifecycle API/event bus constraints | Host adapter interfaces approved before implementation tasks |
| Two modes diverge into two codepaths | Architecture drift; duplicate bugs | Enforce dependency rule: both adapters depend on one core package; no adapter-local protocol conversion | Static dependency check + parity tests pass |
| AUTH_V1 signing drift during language migration | Gateway rejects valid sessions; Phase 1 regression | Build canonical-payload golden fixtures from existing Java logic and verify against gateway integration tests | Plugin-generated signatures accepted by gateway tests |
| Error envelope drift (`AUTH_V1_*`, retry_after behavior) | UI/plugin error handling breaks | Contract tests compare plugin observed responses with doc-defined fields/codes | Contract tests green against docs and gateway behavior |
| Secure credential semantics regress (plaintext/log leaks, wrong scope) | Security incident | Port typed credential errors + redaction tests first, then runtime provider | Secret leakage tests and typed error tests green |
| One-shot cutover removes legacy too early | Delivery stall with no rollback | Delay legacy removal until dual-mode real-chain evidence and parity tests are green | Signed acceptance checklist before deleting main runtime path |
| Build/CI split between Maven and plugin stack | Unverified plugin path in CI | Add explicit plugin test commands to phase plan and CI entrypoint docs | CI runs gateway + plugin contract/integration suites |
