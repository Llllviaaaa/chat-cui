# 01.1 Legacy Cutover Checklist

## Cutover Gate

Cutover from Java `pc-agent` main-path runtime is allowed only if all items below are checked.

## Preconditions

- [x] Host plugin adapter lifecycle/event integration is implemented and tested.
- [x] CLI adapter path is implemented and tested.
- [x] No-drift checks are automated (`verify:no-drift` + CI workflow).
- [x] Acceptance evidence includes Plugin mode and CLI mode proof.
- [x] AUTH_V1 compatibility checks remain green after plugin migration changes.

## Main-Path Cutover Action

- [x] Remove `<module>pc-agent</module>` from root `pom.xml`.
- [x] Keep `pc-agent/` sources in repo for audit/migration reference until dedicated cleanup phase.
- [x] Confirm gateway auth tests still pass after module list update.

## Post-Cutover Validation

- [x] `npm.cmd --prefix pc-agent-plugin run verify:phase-01.1`
- [x] `mvn -pl gateway -Dtest=AuthServiceTest,AuthFailureCodeContractTest,AuthErrorContractDocSyncTest test`
- [x] `mvn -pl gateway -Dtest=AuthEntryIntegrationTest,WsAuthFailureIntegrationTest test`

## Decision

Cutover accepted for phase 01.1. Java `pc-agent` is no longer a primary execution path in root build modules.
