# 01.1 CLI Real-Chain Runbook

## Goal

Run one standalone CLI session against real OpenCode and real AI-Gateway endpoints. This path is `real chain` only and allows `no mocks`.

## Prerequisites

1. Node.js 20+ and project dependencies installed.
2. Reachable OpenCode service endpoint.
3. Reachable AI-Gateway service endpoint with AUTH_V1 credentials provisioned.
4. Secret already stored for the configured `CHATCUI_SECRET_REF`.

## Required Environment

```powershell
$env:CHATCUI_TENANT_ID="tenant-a"
$env:CHATCUI_CLIENT_ID="client-a"
$env:CHATCUI_AK="ak_live_1234"
$env:CHATCUI_SECRET_REF="wincred://tenant-a/client-a"
$env:CHATCUI_CREDENTIAL_STATE="active"
$env:CHATCUI_REAL_CHAIN="true"
$env:CHATCUI_OPENCODE_ENDPOINT="http://127.0.0.1:11400"
$env:CHATCUI_GATEWAY_ENDPOINT="ws://127.0.0.1:8080/ws/skill"
```

## Command

```powershell
npm.cmd --prefix pc-agent-plugin run cli -- run-session --question "hello from real chain"
```

## Expected Output Markers

- `Real chain session started.`
- `OpenCode: ...`
- `AI-Gateway: ...`
- `Session complete. session_id=... trace_id=...`

## Failure Guardrails

- If `CHATCUI_REAL_CHAIN` is not `true`, CLI fails with deterministic AUTH_V1 error signal.
- If `CHATCUI_OPENCODE_ENDPOINT` or `CHATCUI_GATEWAY_ENDPOINT` is missing, CLI fails immediately.
- Missing or disabled credentials fail before runtime session start.
