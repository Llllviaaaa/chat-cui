# 01.1 Architecture Decision Record

## Status

Accepted - 2026-03-03

## Context

Phase 01.1 is an architecture correction phase. The implementation must align with the `message-bridge-opencode-plugin` plugin model and stop using Java `pc-agent` as the primary runtime path.

## Decision

1. `plugin-first` is mandatory.
The runtime is implemented as a PC client plugin adapter layer over one shared bridge core.

2. `dual-mode` runtime is mandatory.
Plugin mode and CLI mode must instantiate the same shared runtime factory and reuse the same bridge/auth contracts.

3. CLI must validate a `real chain`.
The `run-session` command requires `CHATCUI_REAL_CHAIN=true` with both OpenCode and AI-Gateway endpoints configured.

4. `AUTH_V1` compatibility is mandatory.
Canonical signing fields, error code mapping, and secure credential semantics remain compatible with gateway phase-1 contracts.

## Constraints

- Adapter layers (`src/host-adapter`, `src/cli`) cannot own protocol conversion implementation.
- Protocol conversion is owned by `src/core/bridge`.
- Auth signing logic is owned by `src/core/auth`.
- Credential handling is owned by `src/core/security`.

## Consequences

- Host plugin and CLI bootstrap can evolve independently in UX, but not in bridge/auth behavior.
- Architecture drift is blocked by no-drift tests and CI checks in phase 01.1.
- Legacy Java `pc-agent` can be removed from main build path only after dual-mode evidence is produced.
